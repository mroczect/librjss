LIBRJSS(3) Library Functions Manual LIBRJSS(3)

NAME

librjss - Robust async Rust client for Juragan Subsystem (JSS) / Frappe / ERPNext APIs

SYNOPSIS

Cargo dependency

[dependencies]
librjss = { git = "https://github.com/mroczect/librjss.git" }

Import in Rust code

use librjss::{


    RjssClient, ClientConfig, AuthMode, JssError,


    SessionInfo, FrappeBoot, BootUser, BootUserInfo,


    SidebarPages, NavbarSettings, FrequentLink,


    UserInfo, ReportMeta, AppData,
};

DESCRIPTION

librjss provides full access to a JSS (Frappe/ERPNext) server. It supports:

•
Cookie-based session authentication and token-based API key/secret authentication.
•
Automatic CSRF token extraction from the /app dashboard.
•
Configurable retry with exponential backoff.
•
Typed builders for report execution, doctype CRUD, and file upload/download.
•
A rich error model mirroring Frappe JSON error responses.
•
Zero-framework – lightweight dependencies on reqwest, serde, secrecy, and a few other small crates.
•
Unique trace ID per client for distributed tracing.

This manual documents every public module, struct, enum, trait, and method.

CORE CONCEPTS

RjssClient

Central struct holding configuration, HTTP client, session, credentials, and boot data.

Constructor:

RjssClient::new(config: ClientConfig) -> Result<Self, JssError>
Validates configuration, builds a reqwest client with a cookie jar, timeout, and user-agent.

Authentication & session methods:

authenticate(&mut self) -> Result<(), JssError>
Logs in using session or token (depending on AuthMode). In session mode it stores credentials and calls login. Token mode directly constructs a synthetic SessionInfo.
logout(&mut self) -> Result<(), JssError>
Only meaningful for AuthMode::Session. Sends a logout request with the CSRF token, then clears session and credentials.
ensure_session(&mut self) -> Result<(), JssError>
Checks whether the current session is still valid by calling get_logged_user. If a 401 is returned the session is discarded and, if credentials are available, a fresh login is performed automatically.

Accessors:

session_info(&self) -> Option<&SessionInfo>
Returns a reference to the current session data.
boot(&self) -> Option<&FrappeBoot>
Returns the boot data extracted from the /app page after login. Contains user info, permissions, sidebar, etc.
base_url(&self) -> &Url
The base server URL in use.
trace_id(&self) -> &str
Unique UUID v4 identifier for tracing/logging.

Authenticated HTTP requests:

authenticated_get(&self, path: &str) -> Result<String, JssError>
authenticated_post(&self, path: &str, body_json: &str) -> Result<String, JssError>
authenticated_put(&self, path: &str, body_json: &str) -> Result<String, JssError>
authenticated_delete(&self, path: &str) -> Result<String, JssError>
Each sends the corresponding HTTP request to the path joined with base_url. They attach the appropriate authorization header (token or CSRF) and handle retry with exponential backoff.

File operations:

upload_file(&self, file_name: &str, file_content: Vec<u8>, doctype: &str, docname: &str, fieldname: &str) -> Result<String, JssError>
Uploads a file to /api/method/upload_file using multipart/form-data. Returns the raw JSON response.
download_file(&self, file_url: &str) -> Result<Vec<u8>, JssError>
Downloads a file from a relative URL (e.g., "/files/photo.png"). Returns raw bytes.
download_file_to_path(&self, file_url: &str, save_path: &Path) -> Result<(), JssError>
Same as above, but writes directly to disk.

Report builder:

report(&self, name: &str) -> ReportBuilder
Returns a builder for running a named Frappe report.

Doctype / resource builder:

doctype(&self, name: &str) -> ResourceBuilder
Builder for fetching document lists.
get_doc(&self, doctype: &str, name: &str) -> Result<String, JssError>
Fetches a single document by name.
create_doc(&self, doctype: &str, data: &impl Serialize) -> Result<String, JssError>
Creates a new document. data must implement Serialize.
update_doc(&self, doctype: &str, name: &str, data: &impl Serialize) -> Result<String, JssError>
Updates an existing document.
delete_doc(&self, doctype: &str, name: &str) -> Result<String, JssError>
Deletes a document.
call_method(&self, method: &str, args: Option<HashMap<String, String>>) -> Result<String, JssError>
Calls a whitelisted server method (e.g., frappe.email.send) with optional arguments. The body is sent as a JSON POST.

Boot data convenience accessors:

user_info_map(&self) -> Option<&HashMap<String, BootUserInfo>>
sidebar_pages(&self) -> Option<&SidebarPages>
navbar_settings(&self) -> Option<&NavbarSettings>
versions(&self) -> Option<&HashMap<String, String>>
lang_dict(&self) -> Option<&HashMap<String, String>>
frequent_links(&self) -> Option<&[FrequentLink]>
is_developer_mode(&self) -> bool
Returns true when developer_mode != 0 in the boot data.
is_read_only(&self) -> bool
Directly from the read_only field in boot data.

Trait implementation: AuthEndpoints is implemented for RjssClient, allowing static URL builders to be called through the type.

AuthMode (enum)

Authentication strategy selection.

pub enum AuthMode {


    Session {


        email: secrecy::SecretString,


        password: secrecy::SecretString,


    },


    Token {


        api_key: String,


        api_secret: secrecy::SecretString,


    },
}
•
Session – logs in with email/password, manages session cookies, and automatically extracts the CSRF token from /app.
•
Token – sends an Authorization: token api_key:api_secret header on every request. No explicit login or logout is required.

ClientConfig

Configuration container validated at client creation.

pub struct ClientConfig {


    pub base_url: reqwest::Url,


    pub auth_mode: AuthMode,


    pub expected_sitename: Option<String>,


    pub required_roles: Vec<String>,


    pub timeout_secs: u64,          // default 30


    pub max_retries: u32,           // default 3


    pub user_agent: String,


    pub insecure_ssl: bool,         // default false
}

Method:

validate(&self) -> Result<(), JssError>
Ensures HTTPS is used unless insecure_ssl is true, and that email and password are not empty for Session mode.

JssError

Comprehensive error type covering all failure modes.

pub enum JssError {


    Config(String),


    Validation(String),


    Network(reqwest::Error),


    Http { status: StatusCode, body: String },


    ApiError { exc_type: String, message: String, status: StatusCode },


    Auth(String),


    Csrf(String),


    Permission(String),


    SitenameMismatch { expected: String, actual: String },


    NotAuthenticated,


    RateLimited { retry_after: Option<u64> },


    Parse(String),


    Expired,


    Cancelled,


    FileOperation(String),


    Internal(String),
}

Helper methods:

from_api_response(status: StatusCode, body: &str) -> JssError
Parses a Frappe JSON error body. If exc_type is present, returns ApiError; otherwise returns Http.
status_code(&self) -> StatusCode
Maps each variant to an appropriate HTTP status code (e.g., Config → 500, Network → 502, Auth → 401, Csrf → 403, RateLimited → 429).

SessionInfo

Session data stored after a successful login.

pub struct SessionInfo {


    pub sid: secrecy::SecretString,


    pub csrf_token: secrecy::SecretString,


    pub full_name: Option<String>,


    pub sitename: String,


    pub roles: Vec<String>,
}
•
sid – session ID from the login response.
•
csrf_token – token extracted from the /app page.
•
full_name – user’s full name (from boot or login response).
•
sitename – site name from frappe.boot.
•
roles – user role list.

FrappeBoot and related types

Data structure deserialized from the JavaScript frappe.boot object on the /app page. All fields have defaults (mostly Option or empty collections). The full definition can be found in the source; here are the most important fields:

FrappeBoot:

pub user: BootUser,
pub sitename: String,
pub csrf_token: String,
pub sysdefaults: SysDefaults,
pub app_logo_url: Option<String>,
pub home_page: Option<String>,
pub allow_modules: Vec<String>,
pub can_select: Vec<String>,  (and similar permission vectors)
pub all_reports: HashMap<String, ReportMeta>,
pub dashboards: Vec<DashboardMeta>,
pub user_info: HashMap<String, BootUserInfo>,
pub sidebar_pages: SidebarPages,
pub navbar_settings: Option<NavbarSettings>,
pub versions: HashMap<String, String>,
pub lang_dict: HashMap<String, String>,
pub frequently_visited_links: Vec<FrequentLink>,
pub developer_mode: i32,
pub read_only: bool,
...

BootUser: Contains the user’s name, email, full_name, roles, user_type, permissions (a map of doctype to allowed actions), and permission vectors like can_select, can_write, etc.

SysDefaults: default_app: Option<String>, time_zone: Option<String>.

ReportMeta: modified, title, ref_doctype, report_type.

DashboardMeta: name: String.

AppData: app_name, app_title, app_route, app_logo_url, modules, workspaces.

BootUserInfo: fullname, image, name, email, time_zone.

SidebarPages: workspace_setup_completed: Option<i32>, pages: Vec<SidebarPage>, has_access: bool, has_create_access: bool.

SidebarPage: Many fields, see source code.

NavbarSettings: settings_dropdown: Vec<NavbarItem>, help_dropdown: Vec<NavbarItem>.

NavbarItem: item_label, item_type, action, route, hidden, is_standard.

PageInfo: title, modified.

FrequentLink: route: String, count: i32.

AuthEndpoints (trait)

Implemented by RjssClient. Provides static URL builders that can be used without a client instance.

pub trait AuthEndpoints {


    fn login_url(base: &Url) -> Result<Url, JssError>;


    fn logout_url(base: &Url) -> Result<Url, JssError>;


    fn csrf_token_url(base: &Url) -> Result<Url, JssError>;


    fn get_logged_user_url(base: &Url) -> Result<Url, JssError>;


    fn app_page_url(base: &Url) -> Result<Url, JssError>;
}

Each yields URLs such as /api/method/login, /api/method/logout, etc.

ReportBuilder

Built by RjssClient::report(). Allows chaining filters before executing a report.

Methods:

add_filter(mut self, field: &str, value: &str) -> Self
Adds a (field, value) filter. Can be chained.
async run_raw(&self) -> Result<String, JssError>
Runs the report and returns the raw JSON string.
async run<T: DeserializeOwned>(&self) -> Result<T, JssError>
Runs the report and deserializes directly into T.

ResourceBuilder

Built by RjssClient::doctype(). Builder for document listing with filters, field selection, ordering, and pagination.

Methods:

filter(mut self, field: &str, operator: &str, value: &str) -> Self
Adds a filter condition (e.g., filter("status","=","Open")).
fields(mut self, fields: Vec<&str>) -> Self
Specifies which fields to return (JSON array).
order_by(mut self, order: &str) -> Self
Ordering (e.g., "creation desc").
limit(mut self, limit: u32) -> Self
Page size.
limit_start(mut self, start: u32) -> Self
Starting index.
async execute_raw(&self) -> Result<String, JssError>
Executes and returns raw JSON.
async execute<T: DeserializeOwned>(&self) -> Result<T, JssError>
Executes and deserializes into T.
async all<T: DeserializeOwned>(&self) -> Result<Vec<T>, JssError>
Automatically paginates with a page size of 200, collecting all records into a Vec<T>.

Additional public types

UserInfo: { message: UserInfoMessage }, where UserInfoMessage has name, email: Option<String>, roles: Vec<String>.

LoginApiResponse and LoginApiMessage are used internally for login parsing.

USAGE EXAMPLES

1. Setup and session authentication

use librjss::{RjssClient, ClientConfig, AuthMode};
use secrecy::SecretString;
let config = ClientConfig {


    base_url: "https://app.example.com".parse().unwrap(),


    auth_mode: AuthMode::Session {


        email: SecretString::new("user@example.com".into()),


        password: SecretString::new("secret".into()),


    },


    ..Default::default()
};
let mut client = RjssClient::new(config)?;
client.authenticate().await?;
println!("Logged in as: {:?}", client.session_info().unwrap().full_name);

2. Basic GET and POST

let body = client.authenticated_get("/api/resource/ToDo").await?;
let resp = client.authenticated_post(


    "/api/resource/ToDo",


    r#"{"description":"test"}"#
).await?;

3. File upload

let image_bytes = std::fs::read("photo.png")?;
let result = client.upload_file(


    "photo.png", image_bytes, "Customer", "CUST-001", "image"
).await?;
println!("Upload response: {}", result);

4. File download

let bytes = client.download_file("/files/photo.png").await?;
std::fs::write("local.png", bytes)?;
// or directly
client.download_file_to_path("/files/photo.png", Path::new("local.png")).await?;

5. Running a report

#[derive(Deserialize)]
struct MyReportRow { /* ... */ }
let rows: Vec<MyReportRow> = client


    .report("Sales Analytics")


    .add_filter("from_date", "2026-01-01")


    .add_filter("to_date", "2026-06-30")


    .run()


    .await?;

6. Doctype listing with filters

let list: serde_json::Value = client


    .doctype("ToDo")


    .filter("status", "=", "Open")


    .fields(vec!["name", "description", "date"])


    .order_by("creation desc")


    .limit(50)


    .execute()


    .await?;
// Fetch all records
let all: Vec<ToDo> = client


    .doctype("ToDo")


    .filter("status", "=", "Cancelled")


    .all()


    .await?;

7. Single document CRUD

let doc = client.get_doc("Customer", "CUST-001").await?;
#[derive(Serialize)]
struct NewCustomer { customer_name: String, customer_group: String }
let new_cust = NewCustomer { ... };
let created = client.create_doc("Customer", &new_cust).await?;
let updated = client.update_doc("Customer", "CUST-001", &modification).await?;
let deleted = client.delete_doc("Customer", "OLD-CUST").await?;

8. Calling a server method

use std::collections::HashMap;
let mut args = HashMap::new();
args.insert("doctype".into(), "Sales Invoice".into());
args.insert("docname".into(), "SINV-0001".into());
let result = client.call_method("frappe.email.send", Some(args)).await?;

9. Error handling

match client.authenticated_get("/api/resource/ToDo").await {


    Err(JssError::ApiError { exc_type, message, status }) => {


        eprintln!("[{exc_type}] {message} (HTTP {status})");


    }


    Err(JssError::RateLimited { retry_after }) => {


        let secs = retry_after.unwrap_or(5);


        tokio::time::sleep(Duration::from_secs(secs)).await;


        // retry


    }


    Err(e) => {


        eprintln!("Other error: {}", e);


        let status_code = e.status_code();


        // ...


    }


    Ok(data) => { /* process data */ }
}

10. Accessing boot data

if let Some(boot) = client.boot() {


    println!("Site: {}", boot.sitename);


    println!("Roles: {:?}", boot.user.roles);


    for (doctype, perms) in &boot.user.permissions {


        println!("{} can: {:?}", doctype, perms);


    }


    if client.is_developer_mode() {


        println!("Developer mode active");


    }
}

CONFIGURATION (ENVIRONMENT VARIABLES FOR CLI)

The following variables are used by the cli_auth binary and can be adopted in your own applications:

JSS_BASE_URL
Required. Base URL of the JSS instance.
JSS_EMAIL
Email for session auth.
JSS_PASSWORD
Password for session auth.
JSS_TOKEN_API_KEY
API key for token auth.
JSS_TOKEN_API_SECRET
API secret for token auth.
JSS_EXPECTED_SITENAME
Optional. Expected site name; login fails if the boot sitename does not match.
JSS_REQUIRED_ROLES
Optional. Comma-separated required roles.
JSS_TIMEOUT_SECS
Request timeout in seconds (default 30).
JSS_MAX_RETRIES
Maximum retry attempts (default 3).
JSS_USER_AGENT
Custom User-Agent header.
JSS_INSECURE_SSL
Set "true" to accept invalid TLS certificates (e.g., self-signed).

MOCK SERVER FOR DEVELOPMENT

A full mock server is available at examples/jss_mock_server.rs. It replicates the behaviour of a real JSS server:

•
POST /api/method/login → sets cookie + JSON
•
GET /app → HTML with frappe.boot
•
GET /api/method/frappe.auth.get_logged_user
•
POST /api/method/logout
•
GET /api/method/frappe.auth.get_csrf_token (returns 417)

Start it with:

cargo run --example jss_mock_server

Then point the client to http://127.0.0.1:8080 and set JSS_INSECURE_SSL=true.

TESTING

Unit tests (no server required):

cargo test --lib

Integration tests require the mock server to be running. Once the server is up:

cargo test --test client_tests cargo test --test file_tests cargo test --test report_tests cargo test --test resource_tests

Or run everything:

cargo test --workspace
