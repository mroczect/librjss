NAME

librjss − Rust library for cookie-based session authentication

SYNOPSIS

Cargo.toml:
[dependencies]
librjss = { git = "https://github.com/mroczect/librjss.git" }

Rust source:
use librjss::api::AuthManager;
use librjss::handler::config::AuthConfig;
use librjss::handler::session_store::MemorySessionStore;
use librjss::handler::types::{Credentials, UserProvider, SessionStore};
use std::sync::Arc;

DESCRIPTION

librjss is a protocol-agnostic, framework-agnostic Rust library that provides building blocks for cookie-based session authentication. It does not depend on any specific web framework. It defines traits for user lookup and session storage and offers a ready-to-use AuthManager that orchestrates the authentication lifecycle.

The library is structured into two main modules: api (high-level manager and cookie helpers) and handler (lower-level types, configuration, error types, session store implementation, and environment loading).

MODULES

api

api::AuthManager is the central coordinator. It is constructed with an AuthConfig , a UserProvider , and a SessionStore . It exposes methods to login, logout, validate, create, and destroy sessions. api::auth::cookies contains functions to create, parse, and delete session cookies. api::auth::session contains low-level session creation, validation, and destruction logic.

handler

handler::config provides AuthConfig (defaults, builder with validation, SameSite enum). handler::env implements AuthConfig::from_env() to read configuration from environment variables. handler::error defines the AuthError enum with status code mapping and JSON body generation. handler::session_store contains a MemorySessionStore , an in-memory session store implementation suitable for development and testing. handler::types contains all core data types and traits: SessionId, Credentials, UserInfo, SessionInfo, HttpResponse, UserProvider, and SessionStore.

DATA TYPES

SessionId

A newtype wrapper over String representing a session identifier. Construct with SessionId::new(id). Use .as_str() to get a &str. Implements Display, Eq, Hash, Serialize, Deserialize.

Credentials

Holds username: String and password: String used for login.

UserInfo

Returned from UserProvider::authenticate(). Fields: user_id:String, extra: serde_json::Value (arbitrary claims).

SessionInfo

Session record stored in the session store. Fields: user_id:String, data: serde_json::Value , issued_at:OffsetDateTime, expires_at:OffsetDateTime, idle_deadline:Option<OffsetDateTime>.

HttpResponse

A minimal HTTP response decoupled from any framework. Constructors: HttpResponse::new(status,body), HttpResponse::json(status,json_value). Builder: .with_header(key, value) status:http::StatusCode, headers:Vec<(String,String)>, body:String.

TRAITS

UserProvider

async trait that must be implemented for user authentication.
async fn authenticate(&self, creds: &Credentials) -> Result<UserInfo, AuthError>
async fn get_user_by_id(&self, user_id: &str) -> Result<Option<UserInfo>, AuthError>

SessionStore

async trait for session persistence.
async fn save(&self, id: &SessionId, info: &SessionInfo) -> Result<(), AuthError>
async fn load(&self, id: &SessionId) -> Result<Option<SessionInfo>, AuthError>
async fn delete(&self, id: &SessionId) -> Result<(), AuthError>
async fn cleanup(&self) -> Result<(), AuthError>

CONFIGURATION

AuthConfig can be created using defaults, the builder pattern, or from environment.

AuthConfigBuilder methods: .cookie_name(), .cookie_path(), .cookie_domain(), .cookie_secure(), Finalize with .build() -> Result<AuthConfig, AuthError> which validates lifetime > 0 and cookie_name non-empty.

SameSite enum variants: Strict, Lax (default), None.

ENVIRONMENT

AuthConfig::from_env() reads these optional variables (all have defaults):
AUTH_COOKIE_NAME

(default "sid")

AUTH_COOKIE_PATH

(default "/")

AUTH_COOKIE_DOMAIN

(optional)

AUTH_COOKIE_SECURE

"true"/"false" (default "true")

AUTH_COOKIE_HTTP_ONLY

"true"/"false" (default "true")

AUTH_COOKIE_SAME_SITE

"Strict"/"Lax"/"None" (default "Lax")

AUTH_SESSION_LIFETIME

integer seconds (default 86400)

AUTH_SESSION_IDLE_TIMEOUT

integer seconds (optional)

AUTH_LOGIN_URL

optional URL string

AUTH_LOGOUT_REDIRECT_URL

optional URL string

SESSION STORE

MemorySessionStore in-memory implementation for development/testing.
let store = MemorySessionStore::new();
Implements SessionStore with async RwLock<HashMap>. cleanup() removes sessions where expires_at <= now.

AUTHMANAGER API

AuthManager::new(config, user_provider, session_store)
login(credentials) -> Result<HttpResponse, AuthError>

Validates non-empty credentials, calls user_provider.authenticate(), creates session, returns 200 with Set-Cookie and JSON body.

logout(session_id: Option<&SessionId>) -> Result<HttpResponse,
AuthError>

Destroys session (if provided), returns clearance cookie and 200 or redirect.

validate_session(session_id) -> Result<SessionInfo, AuthError>

Checks expiry and idle timeout, deletes expired sessions.

create_session(user_info) -> Result<(SessionId, SessionInfo),
AuthError>

Directly creates a session, returns id and info.

destroy_session(session_id) -> Result<(), AuthError>

Deletes session from store.

COOKIE FUNCTIONS (api::auth::cookies)

create_session_cookie(session_id, config) -> Cookie

Builds cookie with name, path, domain, secure, httpOnly, SameSite, max_age.

parse_session_id_from_cookie(cookie_header, config) ->
Option<SessionId>

Extracts session ID from Cookie header string.

create_removal_cookie(config) -> Cookie

Returns empty cookie with max_age=0 to clear session.

SESSION FUNCTIONS (api::auth::session)

generate_session_id() -> SessionId

32-character alphanumeric random string.

create_session(store, config, user_info) -> Result<(SessionId,
SessionInfo), AuthError>

Generates ID, sets timestamps, saves.

validate_session(store, config, id) -> Result<SessionInfo, AuthError>

Validates and auto-deletes expired sessions.

destroy_session(store, id) -> Result<(), AuthError>

Deletes session.

EXAMPLE

A minimal actix-web integration:
use librjss::api::AuthManager;
use librjss::handler::config::AuthConfig;
use librjss::handler::session_store::MemorySessionStore;
use librjss::handler::types::{Credentials, UserProvider, UserInfo, AuthError};
use std::sync::Arc;

struct MyProvider;
#[async_trait]
impl UserProvider for MyProvider {
async fn authenticate(&self, c: &Credentials) -> Result<UserInfo, AuthError> {
if c.username == "admin" && c.password == "pass" {
Ok(UserInfo { user_id: "admin".into(), extra: serde_json::json!({}) })
} else {
Err(AuthError::InvalidCredentials)
}
}
async fn get_user_by_id(&self, id: &str) -> Result<Option<UserInfo>, AuthError> {
Ok(None)
}
}

let config = AuthConfig::builder().cookie_name("sid".into())
.session_lifetime(time::Duration::hours(2)).build().unwrap();
let manager = AuthManager::new(config, Arc::new(MyProvider), Arc::new(MemorySessionStore::new()));

// In an actix handler:
async fn login(state: web::Data<AppState>, body: web::Json<LoginReq>) -> HttpResponse {
let resp = state.manager.login(Credentials { username: body.user.clone(), password: body.pass.clone() }).await;
// convert librjss::HttpResponse to actix_web::HttpResponse ...
}
